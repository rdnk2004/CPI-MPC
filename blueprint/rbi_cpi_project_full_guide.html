1
Download CPI data from RBI DBIE
Your primary dataset
  • Go to dbie.rbi.org.in → click "Time Series Publications" → "Handbook of Statistics on Indian Economy"
  • Table needed: "Consumer Price Index (Base 2012=100)" — get the combined + rural + urban breakdowns
  • Alternatively: go to rbi.org.in → Publications → Handbook of Statistics → download the Excel for "Prices" section
  • You want monthly data from Jan 2015 to latest available (June 2026)
  • Columns you need: Date, CPI_General, CPI_Food (CFPI), CPI_Fuel, CPI_Core (General minus Food and Fuel)
If DBIE download is slow: MOSPI (mospi.gov.in) also publishes monthly CPI. The Ministry of Statistics data is identical to what RBI uses. Download as Excel, save to your raw_data/ folder.

Download repo rate and MPC decision history

Load and clean in Python — notebook 01
All the wrangling in one place
import pandas as pd import numpy as np # Load CPI data (adjust filename to what you downloaded) cpi = pd.read_excel('raw_data/cpi_rbi.xlsx', sheet_name='CPI', skiprows=4) # Rename columns cleanly cpi.columns = ['date','cpi_general','cpi_food','cpi_fuel', 'cpi_clothing','cpi_housing','cpi_misc'] # Parse dates cpi['date'] = pd.to_datetime(cpi['date']) cpi = cpi.dropna(subset=['date','cpi_general']) cpi = cpi.sort_values('date').reset_index(drop=True) # Compute CPI Core = General minus Food and Fuel (RBI's own definition) cpi['cpi_core'] = cpi['cpi_general'] - ( (cpi['cpi_food'] * 0.4563) + (cpi['cpi_fuel'] * 0.0666) ) # NOTE: weights are from CPI 2012 base year basket # Food weight: 45.63%, Fuel weight: 6.66%, Core: ~47.71% # Compute YoY inflation for each component for col in ['cpi_general','cpi_food','cpi_fuel','cpi_core']: cpi[f'{col}_yoy'] = cpi[col].pct_change(12) * 100 # Load MPC decisions mpc = pd.read_csv('raw_data/mpc_decisions.csv') mpc['date'] = pd.to_datetime(mpc['date']) print(cpi.tail()) print(f"CPI records: {len(cpi)}, MPC records: {len(mpc)}")
The CPI basket weights (food 45.63%, fuel 6.66%) are official RBI/MOSPI numbers from the 2012 base year. Mention these weights in your policy note — it shows you know the methodology, not just the code.

Variance decomposition — the analytical centrepiece
Quantify how much each component contributes to headline inflation
# How much of headline CPI variance is explained by each component? # This is the core analytical finding of the project. components = { 'Food (supply)': cpi['cpi_food_yoy'], 'Fuel (supply)': cpi['cpi_fuel_yoy'], 'Core (demand)': cpi['cpi_core_yoy'] } correlations = {} for name, series in components.items(): valid = cpi[['cpi_general_yoy']].join(series.rename('comp')).dropna() corr = valid['cpi_general_yoy'].corr(valid['comp']) correlations[name] = round(corr, 3) print("Correlation of each component with headline CPI:") for k, v in correlations.items(): print(f" {k}: {v}") # Year-wise average contribution yearly = cpi.groupby('year')[ ['cpi_general_yoy','cpi_food_yoy','cpi_fuel_yoy','cpi_core_yoy'] ].mean().round(2) print("\nYear-wise average inflation by component:") print(yearly) # Save as CSV for your policy note table yearly.to_csv('outputs/yearly_cpi_components.csv')
What you'll typically find: food inflation has higher variance (supply shocks) while core is stickier and more responsive to rate hikes. In 2022–23, fuel dominated. In 2019, food dominated. This is your key finding — rate hikes can only solve the core component.

STL decomposition on core CPI
Separates trend, seasonality, and residual — publishable quality
from statsmodels.tsa.seasonal import STL import matplotlib.pyplot as plt # STL = Seasonal and Trend decomposition using Loess # More robust than classical decomposition — handles outliers well core_series = cpi.set_index('date')['cpi_core_yoy'].dropna() stl = STL(core_series, period=12, robust=True) result = stl.fit() fig, axes = plt.subplots(4, 1, figsize=(14, 10), sharex=True) axes[0].plot(core_series, color='#2c3e50'); axes[0].set_ylabel('Observed') axes[0].set_title('STL decomposition of India core CPI (YoY %)') axes[1].plot(result.trend, color='#e74c3c'); axes[1].set_ylabel('Trend') axes[2].plot(result.seasonal, color='#27ae60'); axes[2].set_ylabel('Seasonal') axes[3].plot(result.resid, color='#95a5a6'); axes[3].set_ylabel('Residual') axes[3].axhline(0, color='black', linewidth=0.5) for ax in axes: ax.grid(True, alpha=0.2) plt.tight_layout() plt.savefig('outputs/03_stl_decomposition.png', dpi=150, bbox_inches='tight') plt.show() # Extract the trend — this is what monetary policy responds to cpi['core_trend'] = result.trend.values

Granger causality test — does core CPI cause rate hikes?
Statistical test with a policy interpretation
from statsmodels.tsa.stattools import grangercausalitytests, adfuller # First: stationarity test (required for Granger) adf_core = adfuller(cpi['cpi_core_yoy'].dropna()) adf_general = adfuller(cpi['cpi_general_yoy'].dropna()) print(f"Core CPI ADF p-value: {adf_core[1]:.4f} " f"({'stationary' if adf_core[1] < 0.05 else 'non-stationary'})") # Merge monthly repo rate into CPI dataframe cpi_mpc = pd.merge_asof( cpi.sort_values('date'), mpc[['date','repo_rate']].sort_values('date'), on='date', direction='backward' ).dropna(subset=['cpi_core_yoy','repo_rate']) # Granger causality: does past core CPI predict future repo rate changes? print("\nGranger causality test: does core CPI Granger-cause repo rate?") gc_data = cpi_mpc[['repo_rate','cpi_core_yoy']].dropna() grangercausalitytests(gc_data, maxlag=6, verbose=True)
If p-value < 0.05 for any lag: you can write "past core CPI values Granger-cause movements in the repo rate, suggesting the MPC responds predictably to demand-side inflation." If not significant: that's also an interesting policy finding — the MPC may be reacting to food/headline, which rate hikes can't fix.

# Key question: In how many MPC hike episodes was food inflation # the PRIMARY driver of headline CPI, not core? # If yes → the hike was responding to a supply shock → ineffective policy # Classify each MPC meeting period merged['food_dominant'] = merged['cpi_food_yoy'] > merged['cpi_core_yoy'] merged['hiking_on_supply_shock'] = ( (merged['is_hike'] == 1) & (merged['food_dominant'] == True) ) hike_episodes = merged[merged['is_hike'] == 1] supply_driven_hikes = merged[merged['hiking_on_supply_shock'] == True] print(f"Total hike decisions: {len(hike_episodes)}") print(f"Hikes where food > core (supply-shock context): {len(supply_driven_hikes)}") print(f"Proportion: {len(supply_driven_hikes)/len(hike_episodes)*100:.1f}%") print() print(supply_driven_hikes[['date','cpi_general_yoy','cpi_food_yoy', 'cpi_core_yoy','bps_change']].to_string())

AI 1
Prophet forecasting — "What will inflation look like in 6 months?"Forecasting
Facebook's time series model — handles seasonality + holidays automatically
  • Prophet is ideal here because Indian CPI has strong seasonality (monsoon → food prices → kharif harvest cycle)
  • You can add Indian-specific holidays/events as regressors (Budget, monsoon onset)
  • Output: 6-month inflation forecast with confidence intervals — a genuinely useful policy product
from prophet import Prophet import pandas as pd # Prophet needs columns named 'ds' (date) and 'y' (value) core_prophet = cpi[['date','cpi_core_yoy']].rename( columns={'date':'ds','cpi_core_yoy':'y'} ).dropna() model = Prophet( yearly_seasonality=True, weekly_seasonality=False, daily_seasonality=False, seasonality_mode='additive', # additive fits CPI better than multiplicative interval_width=0.90 # 90% confidence interval ) # Optional: add Indian macro event as a regressor # model.add_regressor('is_budget_month') model.fit(core_prophet) # Forecast 6 months ahead future = model.make_future_dataframe(periods=6, freq='MS') forecast = model.predict(future) fig1 = model.plot(forecast) fig1.suptitle('Prophet forecast: India core CPI (YoY %) — next 6 months', fontsize=11) import matplotlib.pyplot as plt plt.savefig('outputs/04_prophet_forecast.png', dpi=150, bbox_inches='tight') plt.show() # Component plot — shows seasonal pattern fig2 = model.plot_components(forecast) plt.savefig('outputs/05_prophet_components.png', dpi=150, bbox_inches='tight') plt.show() print("Forecast for next 6 months:") print(forecast[['ds','yhat','yhat_lower','yhat_upper']].tail(6).round(2))
Interview framing: "I used Prophet because it natively handles the kharif harvest seasonal pattern in Indian food prices, which feeds into core through second-round effects. The model gave a 90% confidence interval — I wanted to communicate uncertainty, not false precision, which is important in policy contexts."
AI 2
XGBoost MPC decision classifier with SHAPClassificationXAI
Predict whether the next MPC meeting will be a hike — and explain why
  • Target: is_hike (1 = rate hike, 0 = hold or cut)
  • Features: core CPI, food CPI, fuel CPI, headline CPI, 3-month moving averages, lag values, GDP growth proxy
  • SHAP values tell you which feature pushed the model toward predicting a hike for each meeting
  • This is explainable AI applied to a real monetary policy question — exactly what RBI's AI team would do
import xgboost as xgb import shap from sklearn.model_selection import TimeSeriesSplit from sklearn.metrics import classification_report, confusion_matrix import matplotlib.pyplot as plt # Features feature_cols = [ 'cpi_general_yoy', 'cpi_food_yoy', 'cpi_fuel_yoy', 'cpi_core_yoy', 'core_lag1', 'core_lag2', 'core_3m_avg' ] model_df = merged[feature_cols + ['is_hike']].dropna() X = model_df[feature_cols] y = model_df['is_hike'] # Time series cross-validation — CRITICAL: never use random split for time series tscv = TimeSeriesSplit(n_splits=5) scores = [] for train_idx, test_idx in tscv.split(X): X_train, X_test = X.iloc[train_idx], X.iloc[test_idx] y_train, y_test = y.iloc[train_idx], y.iloc[test_idx] clf = xgb.XGBClassifier( n_estimators=100, max_depth=3, learning_rate=0.1, use_label_encoder=False, eval_metric='logloss', random_state=42 ) clf.fit(X_train, y_train) score = clf.score(X_test, y_test) scores.append(score) print(f"Time-series CV accuracy: {sum(scores)/len(scores):.3f}") print("(Note: ~65 total samples — interpret with caution)") # Final model on all data for SHAP clf_final = xgb.XGBClassifier(n_estimators=100, max_depth=3, learning_rate=0.1, use_label_encoder=False, eval_metric='logloss', random_state=42) clf_final.fit(X, y) # SHAP explanation explainer = shap.TreeExplainer(clf_final) shap_values = explainer.shap_values(X) plt.figure(figsize=(10, 6)) shap.summary_plot(shap_values, X, feature_names=feature_cols, show=False, plot_type='bar') plt.title('SHAP feature importance: what drives RBI rate hike predictions?') plt.tight_layout() plt.savefig('outputs/06_shap_feature_importance.png', dpi=150, bbox_inches='tight') plt.show() # Explain one specific hike decision (e.g., the June 2022 emergency hike) shap.force_plot(explainer.expected_value, shap_values[0], X.iloc[0], matplotlib=True, show=False) plt.savefig('outputs/07_shap_single_decision.png', dpi=150, bbox_inches='tight') plt.show()
Small sample caveat: ~65 MPC meetings is small for ML. Say this in your note: "The model should be interpreted as a pattern-identifier, not a predictive tool. With only 65 observations, it is prone to overfitting — the primary value is in the SHAP analysis revealing which macro signals correlate most strongly with hike decisions." Honesty about limitations is a sign of research maturity and will impress the panel.
Interview framing: "I deliberately chose XGBoost with SHAP over a black-box neural network because in a policy context, explainability matters more than marginal accuracy. A regulator needs to be able to defend why a model flagged something — 'the SHAP value shows core CPI trend was the dominant feature' is defensible. 'The neural net said so' is not."
